# 项目成果与 RAG 流程简述

这个项目是一个面向人工智能专业培养方案、课程资料和教学文档的智能问答系统。它不是普通聊天机器人，而是先从专业资料里检索相关内容，再结合大模型生成回答，并尽量给出来源依据。

目前项目已经完成了前后端和后端核心功能。前端用了 Vue 3、Vite、TypeScript，后端用了 FastAPI，数据部分用了 MongoDB、Redis 和 ChromaDB。整体上已经具备问答、会话、检索、来源展示、登录认证和基础部署能力，不只是一个简单 demo。

RAG 流程大致是：先对 PDF、DOCX 等文档做解析、清洗和分块，再生成向量并写入向量库。用户提问后，系统会先分析问题，再结合关键词检索、向量检索和混合检索召回相关内容，之后通过重排序、CRAG 纠错、Prompt 组装等步骤生成回答，最后再做一层 Self-RAG 反思校验，尽量减少没有依据的回答。

如果后面考虑论文方向，我觉得比较适合往教育场景下的 RAG 优化、自适应检索策略，或者知识图谱和 RAG 结合这几个方向继续深入。
