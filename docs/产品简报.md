# 人工智能专业智能问答系统 - 产品简报

> 创建日期：2026-02-03
> 产品类型：教育科技 / 智能问答
> 技术方案：RAG (检索增强生成)

---

## 一、产品概述

### 1.1 产品名称
**安信工AI小助手** - 基于人工智能专业知识库的智能问答系统

### 1.2 核心价值主张
解决专业知识内容繁杂、检索不便的问题，通过自然语言对话方式，让学生能够快速、准确地获取校规信息。

### 1.3 目标用户
- **主要用户**：在校学生
- **次要用户**：辅导员、教务管理人员
- **潜在用户**：新生、家长

### 1.4 使用场景
| 场景 | 用户需求 | 解决方案 |
|------|---------|---------|
| 快速查询 | "请假流程是什么？" | 直接返回准确的请假规定和步骤 |
| 规则解读 | "旷课会有什么后果？" | 解释相关条款和可能的处罚 |
| 对比查询 | "奖学金和助学金有什么区别？" | 对比两种制度的差异 |
| 常见问题 | "宿舍门禁时间？" | 快速返回具体时间规定 |

## 二、问题分析

### 2.1 现有痛点

| 痛点 | 描述 | 影响 |
|------|------|------|
| **内容分散** | 知识库包含数十个章节，信息分散 | 查找效率低，学生容易错过重要信息 |
| **检索困难** | PDF格式不支持智能搜索，关键词匹配不准确 | 学生难以找到准确答案 |
| **理解成本高** | 条款表述正式、冗长 | 学生需要反复阅读才能理解 |
| **更新不及时** | 规则变更后，学生无法及时获取最新信息 | 导致违规或错过机会 |
| **缺乏个性化** | 无法根据学生身份（年级、专业）推荐相关内容 | 信息过载 |

### 2.2 市场调研（假设）

| 竞品 | 优势 | 劣势 |
|------|------|------|
| 传统PDF手册 | 官方权威、完整 | 检索困难、体验差 |
| 学校官网FAQ | 结构化、分类清晰 | 内容有限、更新慢 |
| 微信群/QQ群 | 互动性强、响应快 | 信息碎片化、难以追溯 |
| 本产品 | 智能检索、自然语言交互 | 需要持续优化准确率 |

## 三、产品功能

### 3.1 核心功能 (MVP)

#### 3.1.1 智能问答
- **功能描述**：用户通过自然语言提问，系统返回基于人工智能专业知识库的准确答案
- **技术实现**：RAG架构 + TF-IDF检索 + 余弦相似度
- **关键指标**：
  - 答案准确率 > 85%
  - 响应时间 < 2秒
  - 支持上下文对话

#### 3.1.2 知识库搜索
- **功能描述**：支持关键词搜索，返回相关条款片段
- **技术实现**：向量检索 + 相似度排序
- **关键特性**：
  - Top-K结果展示
  - 显示相似度分数
  - 支持按章节筛选

#### 3.1.3 会话管理
- **功能描述**：保存用户对话历史，支持多轮对话
- **技术实现**：MongoDB存储 + Session ID管理
- **关键特性**：
  - 会话持久化
  - 历史记录查询
  - 会话删除功能

### 3.2 扩展功能 (V2)

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 个性化推荐 | 根据用户身份推荐相关内容 | P1 |
| 反馈机制 | 用户对答案进行评价，持续优化 | P1 |
| 规则对比 | 对比不同条款的差异 | P2 |
| 通知提醒 | 重要规则变更提醒 | P2 |
| 多语言支持 | 支持英文查询 | P3 |
| 语音交互 | 支持语音输入/输出 | P3 |

### 4.3 数据流程

```
1. 用户提问
   ↓
2. API接收请求
   ↓
3. 会话管理（获取历史）
   ↓
4. 知识检索（TF-IDF + 余弦相似度）
   ↓
5. 答案生成（基于检索结果）
   ↓
6. 返回响应（答案 + 来源 + 相似度）
   ↓
7. 更新会话历史
```

### 10.1 API文档摘要

| 端点 | 方法 | 描述 |
|------|------|------|
| `/` | GET | 健康检查 |
| `/api/chat` | POST | 智能问答 |

### 10.2 数据模型

```python
# 聊天请求
class ChatRequest:
    message: str          # 用户消息
    session_id: str       # 会话ID（可选）

class ChatResponse:
    response: str         # 回答
    session_id: str       # 会话ID
    sources: List[KnowledgeChunk]  # 知识来源
    timestamp: datetime
```
