# 前后端连接问题诊断报告

## 问题总结

发现并修复了以下问题：

### 1. ✅ 已修复：config.py 语法错误
**问题**：第33行 `MIN similarity` 中间有空格
**修复**：改为 `MIN_SIMILARITY`
**影响**：启动时会导致语法错误

### 2. ✅ 已修复：numpy 版本兼容性
**问题**：numpy 2.2.0 与 scikit-learn 1.5.2 可能不兼容
**修复**：降级为 `numpy<2.0.0`
**影响**：可能导致向量计算错误

### 3. ✅ 已修复：缺少 __init__.py
**问题**：backend 目录缺少 `__init__.py`
**修复**：创建了 `backend/__init__.py`
**影响**：Python 无法将 backend 作为包导入

### 4. ✅ 已修复：前端API硬编码
**问题**：API地址硬编码为 `http://localhost:8000`
**修复**：支持环境变量 `VITE_API_BASE_URL`
**影响**：生产环境无法灵活配置

### 5. ✅ 已修复：缺少错误处理
**问题**：前端API调用缺少完善的错误处理
**修复**：添加超时控制、错误消息解析等
**影响**：用户体验差，错误难以定位

### 6. ✅ 已修复：routers/__init__.py 语法错误
**问题**：文件内容只有 `Routers package`，缺少注释符号 `#`
**修复**：添加正确的模块结构和导入
**影响**：后端无法启动

### 7. ✅ 已修复：手册章节点击功能缺失
**问题**：手册章节标签有 `cursor-pointer` 样式但无 `@click` 事件绑定
**修复**：添加 `askAboutItem()` 函数和事件绑定
**影响**：用户点击标签无响应

### 8. ✅ 已修复：缺少 Python 依赖
**问题**：虚拟环境中缺少 jieba, scikit-learn, motor 等依赖
**修复**：安装所有必需依赖包
**影响**：后端无法启动

## 解决网络问题的建议

1. **检查代理配置**：
   ```bash
   unset HTTP_PROXY
   unset HTTPS_PROXY
   ```

2. **使用国内镜像**：
   ```bash
   pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
   ```

3. **升级 pip**：
   ```bash
   python -m pip install --upgrade pip
   ```

4. **更新 Python**：建议升级到 Python 3.10+ 以获得更好的兼容性
